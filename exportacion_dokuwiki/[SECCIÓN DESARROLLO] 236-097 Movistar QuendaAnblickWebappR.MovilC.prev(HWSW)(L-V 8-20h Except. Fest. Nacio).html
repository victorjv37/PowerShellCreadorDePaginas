Instalación en Eclipse (Entorno de desarrollo) 

Configuración del proyecto QuendaCloud en Eclipse: 
 - Windows >> Preferences >> Java >> Installed JREs >> Add JDK 1.8
 - Windows >> Preferences >> Server >> Runtime Environments >> Add Tomcat 8.5
 - proyecto quenda >> properties >> Project Facets >> Dynamic Web Module: 2.5, Java: 1.8, JavaScript: 1.0. Runtimes: apache Tomcat 8.5
 - proyecto quenda >> properties >> Deployment Assembly


* Instalación Quenda:
 - Descargar el código fuente (ruta: https://repo.plexus.services/031-040_multicliente_producto/quenda/), acceder a la carpeta branches/develop, pestaña code y copiamos la url de Clone with HTPPS y la clonamos desde el git. 
   Preguntar al Team Leader cualquier duda.
 - Revisar que el proyecto quenda importado tengan la JDK y versión correcta. Esto se verifica pinchando en el botón derecho sobre el proyecto --> properties --> java build path --> libraries: JavaSE-1.8 Y properties --> java compiler: 1.8
- Hacemos un project clean del proyecto quenda y si no se queda limpio, en los casos de los que den error de librerías hay que buscar en el proyecto a ver si están ahí, Si no están, solicitarlos a algún compañero del equipo 
  para que os lo pase.
- Una vez añadido la configuración hacer un project clean del proyecto quenda.
- Recordar después de hacer esos cambios, que quizás haya cambiado la versión que se está usando de java. Ir al Java Build Path y comprobar.
- Se crea un nuevo servidor Tomcat 8.5 en el Eclipse, se añaden el proyecto al servidor y se reinicia.
- Si todo ha ido bien se accede a la aplicación a través de la ruta: http://localhost:8080/quenda/
